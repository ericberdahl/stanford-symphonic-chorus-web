Welcome

How to update the SSC website - Overview

To update this website, you must have a Stanford ID and be authorized to access the hosting server. We use a text editor and FileZilla or Dreamweaver to update and upload changes, taking care to test and correct any errors before uploading.

Before you update, download all the files, including image, css, and js folders, and make all changes on your own machine, testing and reviewing them locally before you update the Stanford web server.

What to Update

We need to update the following pages every quarter:

index.html
The index page is the home page for the SSC site and contains an introduction to the chorus and information about upcoming events. Every quarter the items to update are the events in the events section, and the information about the next concert in the middle column. The section titled “The Chorus” at the top of the middle columm is generally stable (although it was modified for Steve Sano’s sabbatical).
performances.html
The performances page is a repository of information about past performances as well as the immediately upcoming concert. This is where we link images of concert posters, Steve’s “For your listening pleasure” (FYLP.html) documents, and list information about performance dates, soloists, and conductors. Add files specific to an upcoming quarter in a new folder for the quarter in the “performances” folder using the naming convention qYYYY, where q is a capital letter for the quarter (F for Fall, W for Winter, and S for Spring), and YYYY is the year, such as 2014.
MemberPage.html
The member information page is updated each quarter to add the current syllabus and registration links. The syllabus is stored in the “performances” folder for the quarter. The member information page also links the rehearsal schedule for the current quarter, and contains any updated audition information that may be specified.
schedule.html
The schedule page gives details of the current quarter and upcoming season for chorus members. Each quarter, update data about current repertoire, rehearsals, and concert calls, based on information provided in the syllabus, and shift information out of the “Balance of the Season” section.

Other pages—about.html, contact.html, and nofragrance.html—are generally stable and do not require regular quarterly updates, only changes as needed.

General Advice

The list items below describe changes that apply to all pages. Click on the links to other pages for how to update those specific pages.

  1. Stanford Templates. All pages use 2010 Stanford University templates that were modified to allow more flexibility in your base text size. A prominent feature of these templates is a layout of multiple columns of fixed widths. Each column is contained in a div section with an id attribute that identifies it to give it its CSS properties, as follows:
    id=“content_main”
    The main content, containing columns identified as content_left and content_center.
    id=“content_left”
    The left-most column for event information (in <div id="events">) and links specifically for SSC members.
    id=“content_center”
    The middle column for the main content of the page.
    id=“sidebar”
    The right-most column for helpful links.
  2. Folders. This website is organized into the following folders (some of which are standard, such as css, js, and images):
    Archive
    A place to save old, unused source files
    css
    All CSS files. To improve the accessibility of this site, all formatting instructions are kept in CSS files in this folder.
    images
    All images (gif, jpg, png, etc.)
    js
    Files for JavaScript functions
    performances
    Contains a folder for each quarter with information specific to that quarter, and sometimes backups of the main files for that quarter. Many of these files will continue to be referenced in the performances.html file.
  3. Case-specific filenames. The Stanford servers are UNIX systems, so that all filenames must be case-specific. Be sure that any file you add is referenced exactly as it is stored. For example, if you have an image file pretend.JPG, then you may not reference it using <img src=“images/Pretend.jpg”. You must reference it using <img src=“images/pretend.JPG”.
  4. Relative paths. In a link, the path to a file in the SSC website must be relative to the folder containing the file with the link. Only files to external sites begin with http://.
  5. Expanding thumbnail images. When you click on most thumbnail images, a JavaScript library opens a larger view of the image in a floating window, called a “mediabox.” Special attributes and an anchor tag surrounding the image tag make this happen. Consider, for example, R. A. Wilson’s image of the Memorial Church that appears in the left column of the home page; it is coded like this: 
    <a class="photo" 
   href="../images/graphics/MemChu.jpg"
   title="Memorial Church, 22 February 2010. Photo by R. A. Wilson."
   rel="mediabox[867 562]"> 
   <img class="framed"
        src="../images/graphics/MemChu.jpg" width="149" height="77"
        alt="Memorial Church evening. Photo by R. A. Wilson." />
</a>
    1. An a (anchor) tag surrounds the img tag. This creates a link that opens the mediabox when you click on it.
    2. The a tag must include the attribute class="photo".
    3. The href attribute of the a tag locates the full-sized image.
    4. The title attribute of the a tag gives the text that appears when you hover with a mouse over the link.
    5. The rel attribute of the a tag specifies the size, in pixels, of the mediabox. Its value has the format mediabox[width height] .
    6. The content of the link is the thumbnail, given in an img tag.
    7. The class attribute of the img tag is the name of a class selector in the css/custom.css that specifies the shadowy frame of the thumbnail. By convention, the class name corresponds to the size of the thumbnail. In actuality, however, the frame image is 15 pixels wider and 15 pixels taller than the thumbnail inside it. To create a frame for a thumbnail image of a given size, first check the css/custom.css file for existing class selectors. If the right size is not found, copy one and modify its name and the name of the image file that contains the frame image. Then copy the frame image file and edit it to actually make it the right size.
    8. The src attribute of the img tag locates the thumbnail image.
    9. The width and height attributes of the img tag give the size of the thumbnail.
    10. The alt attribute of the img tag gives text that appears in place of the thumbnail if the source file cannot be found.
  6. Hiding email addresses. Email addresses are constructed using a JavaScript function named unScramble so that web crawlers do not recognize the email address to put it on junk mailing lists. The unScramble function is in the js/emus.js file. This function takes the following arguments:
    eMail1
    Name to email, such as Webmaster.
    eMail2
    Domain to email, with an extended top-level domain, such as erictuan.commuter, stanford.educator, or erictuan.organizer. Only the first three characters of the top-level domain are used.
    displayText
    Displayed name, such as Eric Tuan.
    subjectText
    Default subject line, such as SSC Website feedback.
    Call unScramble inside a script section and include a manually scrambled email address in the following noscript section for the benefit of users who have disabled JavaScript. For example: 
    <script type="text/javascript">
unScramble("SSCWebMaster", "liztomsharp.commuter",
           "Liz and Tom Sharp", "SSC Website feedback");
</script>
<noscript>Liz & Tom Sharp: WebMaster/AT/liztomsharp/DOT/-c-o-m-/</noscript>
    An email address for the webmaster is included at the bottom of each page.
  7. Publication dates. Meta tags in every file include the publication dates. Updating these help tell search engines to refresh their caches. Look for these and update them each quarter: 
    <meta name="publication_date" content="2014-08-15" />
<meta name="DC.date.issued" content="2014-08-15" />
  8. Webmaster link in the footers. We have given an email address to identify ourselves as webmasters. This name and email address should be updated whenever someone else assumes responsibility for the website.
  9. Notes on Style
    1. Write dates in the format day Month year, such as 15 March 2014. Never abbreviate the month except in the events section on the home page.
    2. Write times with a 12-hour clock followed by the meridiem: a.m. or p.m. Always lower-case the meridiem.
